Problem 1: Using induction to prove algorithm correctness

Given what I saw when grading the last assignment, I think there is benefit in spending a little more time working on proof by induction. Think of this as practice for the exam!

1.1 DLN 5.71 with the following two modifications to the problem.

First, I want you to imagine that line 3 of the algorithm has been modified so it says:

$middle := \text{chooseRandom}(1,n)$

where $\text{chooseRandom}(a,b)$ is a function that returns a randomly chosen number between $a$ and $b$ (inclusive of $a$ and $b$).

Second, decide whether this change to the algorithm preserves the correctness or breaks it. Either way, prove your answer.

Note: on lines 7 and 9, you will see statements like $A[a…b]$ where $a$ and $b$ are numbers. This should be intrepreted as producing a brand-new array of length $b-a+1$ that includes elements $A[a]$, $A[a+1]$, …, $A[b-1]$, $A[b]$ in that order.

Example: suppose $A := [3, 12, 16, 28, 13, 29]$ and consider

$B := A[3..5]$

Then $B = [16, 28, 13]$. (Note the indexing starts at 1.)
